# cn-main-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面内容相互独立，不隶属于任何特定网站或域名。仓库主要作为静态页面的存储与分发点，方便用户直接访问或引用。

## 项目简介

cn-main-pages 是一个通用型 HTML 页面存档仓库。其目的并非托管完整应用或动态服务，而是将一系列静态 HTML 文件集中存放，供需要时直接使用。仓库内容可能包含信息展示页、功能工具页、说明文档页等各类独立页面。

## 目录说明

```
cn-main-pages/
├── index.html          # 仓库入口页面（如存在）
├── pages/              # 存放具体 HTML 页面文件的目录
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 页面所需的静态资源（如 CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：存放所有独立 HTML 页面文件，每个文件为一个完整页面。
- **assets/**：存放页面引用的辅助资源文件，按类型分目录管理。

## 页面归档说明

- 每个 HTML 页面均为自包含文件，尽量不依赖外部资源；若需引用资源，应置于 `assets/` 目录下。
- 页面命名应简洁明了，建议使用英文小写字母、数字与连字符，避免特殊字符。
- 归档的页面不限制主题与用途，但应遵守法律法规和 GitHub 社区规范。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面内容。
- 页面内容如有变动，会通过 Git 提交记录体现，建议使用者关注提交历史。
- 如需引用本仓库中的页面，建议使用 GitHub Pages 或 raw 链接，但请注意链接稳定性。
- 欢迎通过 Issue 提出建议或报告问题，但不保证及时响应。

## 其他

- 本仓库不保证页面内容的长期可用性，维护者有权随时调整或删除页面。
- 本仓库不收集用户信息，也不包含任何跟踪代码。
